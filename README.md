# EduTesting

EduTesting — веб-сервис онлайн-тестирования студентов на `Django`.
Проект сочетает простой интерфейс на `Django Templates` и сильную серверную часть:
автоматическую проверку попыток, аналитику по темам, персональные условия, апелляции,
уведомления, достижения, контроль подозрительных попыток, меры безопасности и
полноценную административную панель.

## Ключевые возможности

### Для студента

- запись на курс
- прохождение тестов с ограничением по времени и числу попыток
- автосохранение черновика попытки
- подробный результат с разбором ответов
- сравнение с прошлой попыткой
- достижения по результатам обучения
- подача апелляции
- получение уведомлений

### Для преподавателя

- создание и публикация курсов, тестов, вопросов и объявлений
- просмотр завершенных попыток
- комментарии к попыткам
- индивидуальные условия по тестам
- аналитика по темам и студентам
- список студентов, которым нужно внимание
- контроль подозрительных попыток
- экспорт результатов курса в `CSV`

### Безопасность и администрирование

- пароли хранятся только в виде криптографического хеша
- включены стандартные валидаторы паролей Django
- используется усиленное хеширование через `ScryptPasswordHasher`
- после серии неудачных попыток вход временно блокируется
- защищены cookie и включены дополнительные security headers
- есть web-страница смены пароля и API-эндпоинт смены пароля
- доступна полноценная панель `Django Admin` по адресу `/admin/`

## Архитектура

Проект построен по паттерну `MVT`.

- `accounts/` — пользователи, роли, формы входа, профиль и безопасность аккаунта
- `testing/models.py` — предметные сущности: курсы, тесты, попытки, апелляции, уведомления, override-условия
- `testing/views.py` — web-flow студента и преподавателя
- `testing/services.py` — прикладные сценарии: проверка попытки, автосохранение, уведомления
- `testing/analytics.py` — диагностика по темам, рейтинг, достижения, integrity-контроль
- `testing/api_views.py` и `testing/api_serializers.py` — REST API и OpenAPI-описание
- `templates/` — клиентское представление на серверных шаблонах

## Что хранится в базе данных

В `db.sqlite3` хранится вся предметная модель сервиса.

Основные сущности:

- `User` — учетная запись, роль, почта, группа и профиль пользователя
- `Course` — учебный курс и его описание
- `Enrollment` — запись студента на курс
- `Announcement` — объявления внутри курса
- `Quiz` — тест и его настройки
- `Question` — вопрос теста
- `Choice` — вариант ответа
- `Attempt` — попытка прохождения теста
- `AttemptAnswer` — ответы внутри попытки
- `AttemptReview` — комментарий преподавателя по попытке
- `AttemptDraft` — автосохраненный черновик попытки
- `AttemptAppeal` — апелляция по попытке
- `UserNotification` — уведомления пользователя
- `QuizAccessOverride` — индивидуальные условия по времени и попыткам

## Быстрый запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_demo_data
python manage.py runserver
```

После запуска доступны:

- web-интерфейс: `http://127.0.0.1:8000/`
- админ-панель: `http://127.0.0.1:8000/admin/`
- Swagger UI: `http://127.0.0.1:8000/api/docs/`
- OpenAPI schema: `http://127.0.0.1:8000/api/schema/`

## Перспективы развития

- переход с `SQLite3` на `PostgreSQL` и подготовка production-конфигурации для развертывания на сервере
- вынесение фоновых задач в очередь для уведомлений, отложенных публикаций и периодических отчетов
- расширение security-блока: журнал действий администратора, подтверждение критичных изменений и более гибкая политика паролей
- импорт тестов и вопросов из `CSV` / `XLSX` с предварительной валидацией структуры данных
- персональные рекомендации студенту на основе слабых тем, прошлых попыток и динамики прогресса
- развитие преподавательской аналитики: сводки по группам, проблемным темам и студентам зоны риска
- интеграция с внешними системами авторизации и учебными платформами
