Полноценный Django-проект по теме **«Веб-сервис онлайн-тестирования студентов»**.

Проект выполнен как курсовая работа с акцентом на:

## 1. Назначение проекта

`EduTesting` предназначен для автоматизации контроля знаний студентов в рамках учебных дисциплин.

Система позволяет:

- публиковать курсы и тесты;
- записывать студентов на курсы;
- автоматически проверять попытки;
- хранить результаты тестирования;
- анализировать успеваемость по курсам, тестам и темам;
- формировать преподавательскую обратную связь;
- выгружать журнал результатов по курсу в `CSV`.

Проект ориентирован именно на учебную демонстрацию backend-разработки на `Django`.

## 2. Основные возможности

- регистрация и авторизация пользователей;
- роли `студент` и `преподаватель`;
- создание, публикация и редактирование курсов;
- создание тестов, вопросов и вариантов ответов;
- запись студента на курс по коду доступа;
- прохождение тестов с лимитом по попыткам;
- автоматическая проверка single-choice и multiple-choice вопросов;
- хранение времени прохождения, баллов и процента результата;
- преподавательская аналитика по курсу и тесту;
- диагностика по темам с рекомендациями;
- рейтинг студентов внутри курса;
- комментарий преподавателя к попытке;
- экспорт результатов курса в `CSV`;
- API с токен-авторизацией для демонстрации backend-части через Swagger и Postman;
- student API-сценарии: запись на курс, старт и отправка попытки;
- teacher API-сценарии: просмотр аналитики и попыток по тесту;
- Swagger UI для визуальной проверки API;
- автотесты проекта.

## 8. Структура данных

Основные модели базы данных:

### `User`

Расширенная пользовательская модель:

- логин;
- email;
- имя и фамилия;
- роль;
- учебная группа;
- краткая информация о пользователе.

### `Course`

Содержит:

- название дисциплины;
- код дисциплины;
- описание;
- семестр и учебный год;
- даты начала и окончания;
- политику оценивания;
- код доступа к курсу;
- статус публикации.

### `Enrollment`

Связывает студента и курс, хранит статус записи.

### `Quiz`

Содержит параметры теста:

- лимит времени;
- проходной балл;
- число попыток;
- сроки доступности;
- настройку показа правильных ответов.

### `Question`

Содержит:

- текст вопроса;
- тему;
- пояснение;
- тип вопроса;
- сложность;
- баллы;
- порядок отображения.

### `Choice`

Хранит варианты ответа и флаг правильности.

### `Attempt`

Фиксирует:

- студента;
- тест;
- статус;
- время начала и завершения;
- длительность;
- набранные баллы;
- процент результата;
- число правильных ответов.

### `Answer`

Хранит ответ на конкретный вопрос в рамках конкретной попытки.

### `Announcement`

Содержит объявления преподавателя по курсу.

### `AttemptReview`

Хранит ручной комментарий преподавателя:

- попытку;
- преподавателя;
- текст комментария;
- дату проверки.

Для преподавателя рассчитывается рейтинг на основе:

- лучших результатов по тестам;
- числа завершенных тестов;
- общего прогресса по курсу.

Это позволяет быстро увидеть лидеров и студентов, которым нужно усиление.

Преподаватель может выгрузить журнал курса в `CSV`, где есть:

- список студентов;
- статус записи;
- средний и лучший результат;
- дата последней попытки;
- результаты по каждому тесту курса.
