## 1. Назначение проекта

`EduTesting` предназначен для автоматизации контроля знаний студентов в рамках учебных дисциплин.  
Система позволяет:

- публиковать курсы и тесты;
- организовывать доступ студентов к дисциплинам;
- автоматически проверять попытки;
- хранить результаты тестирования;
- анализировать успеваемость;
- выявлять проблемные темы и студентов, которым требуется внимание преподавателя.

Проект ориентирован на учебную среду и подходит для демонстрации:

- работы паттерна `MVT`;
- серверной логики на `Django`;
- проектирования базы данных на `SQLite3`;
- тестирования backend-сценариев;
- базового API-документирования через `Swagger`.

## 2. Основные возможности

- регистрация и авторизация пользователей;
- роли `студент` и `преподаватель`;
- создание и редактирование учебных курсов;
- публикация тестов, вопросов и вариантов ответов;
- запись студентов на курс по коду доступа;
- прохождение тестов с ограничением по попыткам;

- хранение результатов, времени и статистики попытки;
- просмотр аналитики по курсу, тесту и студентам;
- диагностика по темам с рекомендациями;
- минимальный read-only API для демонстрации backend-части;
- Swagger UI для визуальной проверки API;
- автоматические тесты проекта.

## 8. Структура данных

Основные модели базы данных:

### `User`

Расширенная пользовательская модель:

- логин;
- email;
- имя и фамилия;
- роль;
- учебная группа;
- краткая информация о пользователе.

### `Course`

Хранит:

- название дисциплины;
- код дисциплины;
- краткое и полное описание;
- аудиторию;
- семестр;
- учебный год;
- сроки;
- политику оценивания;
- код доступа к курсу;
- признак публикации.

### `Enrollment`

Связывает студента и курс.

### `Quiz`

Хранит параметры теста:

- описание;
- инструкции;
- лимит времени;
- проходной балл;
- число попыток;
- доступность по датам;
- настройку показа правильных ответов.

### `Question`

Содержит:

- текст;
- тему;
- пояснение;
- тип вопроса;
- уровень сложности;
- баллы;
- порядок показа.

### `Choice`

Хранит варианты ответа и признак правильности.

### `Attempt`

Фиксирует:

- студента;
- тест;
- статус;
- время начала и завершения;
- длительность;
- набранные баллы;
- процент результата;
- число правильных ответов.

### `Answer`

Хранит ответ на конкретный вопрос в рамках конкретной попытки.

### `Announcement`

Содержит объявления преподавателя по курсу.

## 11. Установка и запуск

### Быстрый запуск

Из корня проекта:

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_demo_data
python manage.py runserver
```

После запуска проект будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

### Если сервер уже был запущен

Остановите его:

```bash
Ctrl + C
```

Затем снова выполните:

```bash
python manage.py runserver
```

## 16. Диагностика и аналитика

В проект добавлена backend-фича, усиливающая курсовую работу:

### Диагностика по темам

После прохождения теста система:

- определяет сильные и слабые темы;
- анализирует точность по темам;
- формирует рекомендации для повторения;
- показывает студенту, что лучше повторить.

### Диагностика курса

Для преподавателя доступны:

- проблемные темы курса;
- число студентов по каждой теме;
- сводка по качеству усвоения;
- список студентов, которым требуется внимание.
