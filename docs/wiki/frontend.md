# Клиентский слой

## Общий подход

Клиентский слой построен на `Django Templates`.

Цель была в том, чтобы интерфейс:

- оставался простым;
- не требовал отдельного frontend-приложения;
- выглядел собранно и современно;
- поддерживал понятную межстраничную навигацию.

## Основные страницы

- главная страница;
- каталог курсов;
- рабочая панель пользователя;
- страница курса;
- диагностика курса;
- страница теста;
- форма прохождения попытки;
- результат попытки;
- список попыток по тесту;
- центр уведомлений;
- кастомная страница Swagger UI.

## Межстраничная навигация

Навигация построена вокруг основных сценариев:

- вход и профиль;
- переход в каталог курсов;
- доступ к рабочей панели;
- переход к курсу, тесту и результату;
- отдельный маршрут для уведомлений;
- отдельный маршрут для API-документации.

## Принципы интерфейса

- без перегрузки визуальными элементами;
- акцент на понятных действиях пользователя;
- минимум JavaScript, кроме действительно полезных сценариев вроде автосохранения;
- единый шаблон и повторяемая структура страниц.

## Где лежит клиентский слой

- [templates/base.html](../../templates/base.html) — базовый шаблон и общая навигация;
- [templates/testing/](../../templates/testing) — страницы приложения;
- [static/css/](../../static/css) — стили интерфейса и Swagger UI.

## Что важно для релизной проверки

- в шаблонах используется `<meta charset="utf-8">`;
- русские тексты отображаются корректно;
- новые backend-фичи не превратили интерфейс в перегруженную панель.
