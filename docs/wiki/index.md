# Wiki EduTesting

Wiki помогает быстро пройтись по системе от предметной области до релизной проверки.

## Разделы

- [Предметная область](domain.md) — задачи сервиса, роли пользователей, ключевые сценарии и ограничения.
- [Архитектура](architecture.md) — как проект разложен по MVT, приложениям и потокам данных.
- [Серверная логика](backend.md) — бизнес-сценарии, сервисный слой, аналитика и вычисляемые backend-фичи.
- [Слой базы данных](database.md) — основные сущности, связи и причины выбора SQLite3 + Django ORM.
- [Клиентский слой](frontend.md) — межстраничная навигация, шаблоны и принципы простого интерфейса.
- [API и интеграции](api.md) — REST API, Swagger UI, Postman и демонстрационные сценарии.
- [Релизная проверка](release-checklist.md) — чеклист перед релизом и соответствие ключевым требованиям.

## Быстрый маршрут

Если нужно быстро показать систему:

1. Прочитать [Предметную область](domain.md).
2. Открыть [Архитектуру](architecture.md).
3. Перейти в [API и интеграции](api.md).
4. Свериться с [Релизной проверкой](release-checklist.md).
