# Демонстрация API в отчёте

## Назначение

В проект добавлен минимальный вспомогательный API для демонстрации работоспособности сервиса через `Swagger` и `Postman`.
Он не заменяет основной веб-интерфейс на `Django Templates`, а используется как дополнительный слой для тестирования и презентации курсовой работы.

## Доступные адреса

- `GET /api/stats/` — сводная статистика по сервису
- `GET /api/courses/` — список опубликованных курсов
- `GET /api/courses/<id>/` — детальная информация о курсе
- `GET /api/quizzes/<id>/` — детальная информация о тесте и вопросах
- `GET /api/schema/` — OpenAPI-схема
- `GET /api/docs/` — Swagger UI

После запуска проекта полный адрес Swagger:

```text
http://127.0.0.1:8000/api/docs/
```

## Что можно показать в отчёте

Для демонстрации работоспособности API достаточно 4-5 скриншотов:

1. Главная страница Swagger с перечнем доступных методов.
2. Выполнение `GET /api/courses/` через Swagger или Postman.
3. Выполнение `GET /api/courses/1/` с детальной структурой курса.
4. Выполнение `GET /api/quizzes/1/` с вопросами и вариантами ответов.
5. Выполнение `GET /api/stats/` со сводной статистикой системы.

## Как запустить

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_demo_data
python manage.py runserver
```

## Краткий комментарий для отчёта

В пояснительной записке можно указать:

> Для дополнительной демонстрации работоспособности серверной части в проекте реализован минимальный REST API. Тестирование API выполнялось через Swagger UI и Postman. Данный API носит вспомогательный характер и не усложняет основную архитектуру веб-приложения, построенную на Django MVT и Django Templates.
