## 1. Назначение системы

EduTesting — веб-сервис для проведения онлайн-тестирования студентов.

Система решает несколько прикладных задач:

- публикация курсов и тестов
- автоматическая проверка результатов
- хранение истории попыток
- аналитика по темам и успеваемости
- персональная обратная связь для студента
- инструменты преподавателя для контроля группы
- административное управление пользователями и учебным контентом

## 2. Предметная область

В образовательном процессе преподавателю важно не только выдать тест, но и:

- понимать, какие темы проседают
- видеть прогресс конкретного студента
- выделять случаи, где нужен ручной просмотр
- управлять индивидуальными условиями прохождения

Студенту, в свою очередь, важно:

- проходить тест без потери ответов
- видеть подробный разбор результата
- понимать слабые темы
- получать понятную обратную связь и видеть личный прогресс

## 3. Выбранные технологии

- `Python` — основной язык разработки
- `Django` — web-фреймворк
- `Django ORM` — работа с базой данных
- `SQLite3` — встроенная СУБД
- `Django Templates` — клиентский слой без отдельного SPA
- `Django REST Framework` — REST API
- `drf-spectacular` — OpenAPI schema и Swagger UI

## 4. Архитектура

Проект построен по паттерну `MVT`.

- `Model` — предметные сущности, ограничения и вычисляемые свойства
- `View` — обработка web- и API-запросов
- `Template` — простой HTML-интерфейс

Дополнительно серверная логика разделена по слоям:

- `services.py` — сценарии выполнения действий
- `analytics.py` — диагностические и рейтинговые вычисления
- `api_serializers.py` / `api_views.py` — внешний API

## 6. Реализованные возможности

### Базовый функционал

- регистрация и авторизация пользователей
- роли преподавателя и студента
- создание курсов и тестов
- прохождение тестов с ограничением по времени
- автоматическая проверка результатов
- история попыток
- комментарии преподавателя
- экспорт результатов в `CSV`

### Усиленные backend-сценарии

- автосохранение черновика попытки
- сравнение текущей попытки с предыдущей
- диагностика по темам
- рейтинг студентов по курсу
- уведомления о важных событиях
- индивидуальные условия по тесту
- апелляции по попыткам
- вычисляемые достижения студента
- контроль подозрительных попыток

### Безопасность и административный контур

- пароли хранятся только в виде хеша
- используется усиленное хеширование через `ScryptPasswordHasher`
- включены стандартные валидаторы паролей Django
- реализована временная блокировка после серии неудачных попыток входа
- доступны защищенная смена пароля через web и API
- есть полноценная панель `Django Admin` для управления системой

## 7. Контроль подозрительных попыток

В системе реализован отдельный вычисляемый слой контроля честности.

Для завершенной попытки автоматически формируются флаги внимания, если обнаружен один
из признаков:

- идеальный результат за короткое время
- слишком быстрое завершение при высоком результате
- резкий скачок между соседними попытками

Эти сигналы:

- показываются преподавателю на дашборде
- входят в диагностику курса
- выводятся в журнале попыток
- доступны через API `GET /api/courses/{id}/integrity/`

## 8. Достижения студента

Достижения вычисляются на лету из уже существующих попыток и не перегружают схему БД.

Примеры достижений:

- первая завершенная попытка по курсу
- результат `100%`
- серия успешных прохождений
- сильный прогресс между попытками

Они отображаются:

- на рабочей панели студента
- на странице результата попытки
- через API `GET /api/my/achievements/`

## 9. Клиентский интерфейс

Интерфейс намеренно оставлен простым:

- `Django Templates`
- минимальный JavaScript только там, где он действительно нужен
- панели, таблицы и карточки без SPA-слоя
- адаптация под новые backend-фичи без усложнения UX

В уже существующие страницы встроены:

- безопасность аккаунта
- быстрый доступ в админку для staff-пользователей
- блоки с подозрительными попытками, достижениями и апелляциями

## 10. API

В проекте реализован REST API для демонстрации ключевых серверных сценариев независимо
от web-интерфейса.

Ключевые сценарии:

- токен-авторизация
- смена пароля
- запись на курс
- старт попытки
- автосохранение черновика
- отправка попытки
- просмотр результата
- получение достижений
- аналитика курса
- контроль подозрительных попыток
- выдача индивидуальных условий
- работа с апелляциями

Для API подготовлены:

- Swagger UI
- OpenAPI schema
- Postman collection
- Postman environment

## 11. Тестирование

Проект покрыт автоматическими тестами.

Проверяются:

- модели и формы
- сервисы проверки попыток и автосохранения
- аналитика по темам, достижения и integrity-контроль
- web-сценарии студента, преподавателя и администратора
- права доступа и безопасность входа
- API, Swagger и схема OpenAPI

Запуск:

```bash
python manage.py test
```
