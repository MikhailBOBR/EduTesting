# Пользовательское описание EduTesting

Этот файл объясняет, как пользоваться приложением EduTesting без привязки к внутреннему коду. Он подходит для первого знакомства со системой, демонстрации проекта и помощи пользователям, которые открывают сервис впервые.

## Что такое EduTesting

EduTesting — это веб-приложение для онлайн-тестирования студентов. В нем преподаватель создает курсы, тесты, вопросы и объявления, а студент записывается на курсы, проходит тесты, получает результаты и может подать апелляцию.

Приложение работает через обычный браузер. После запуска проекта основной интерфейс доступен по адресу:

```text
http://127.0.0.1:8000/
```

## Роли пользователей

В системе есть две основные роли.

**Студент** проходит обучение и тестирование:

- регистрируется или входит в аккаунт;
- просматривает каталог курсов;
- записывается на курс;
- запускает доступные тесты;
- отвечает на вопросы;
- отправляет попытку на проверку;
- смотрит подробный результат;
- получает уведомления;
- при необходимости отправляет апелляцию.

**Преподаватель** управляет учебным процессом:

- создает и редактирует курсы;
- добавляет тесты, вопросы и варианты ответов;
- публикует объявления;
- смотрит попытки студентов;
- оставляет комментарии к результатам;
- выдает индивидуальные условия по тесту;
- анализирует результаты курса;
- рассматривает апелляции.

Администратор также может управлять данными через стандартную панель Django Admin по адресу `/admin/`.

## Первый вход

Для демонстрации можно использовать готовые аккаунты после загрузки демо-данных:

```text
Преподаватель: teacher_demo / TeacherDemo123!
Студент: student_demo / StudentDemo123!
Администратор: admin_demo / AdminDemo123!
```

Если аккаунта нет, пользователь может открыть страницу регистрации, заполнить данные и выбрать подходящую роль.

## Основная навигация

В верхнем меню доступны основные разделы приложения:

- `Главная` — стартовая страница сервиса;
- `Курсы` — список опубликованных курсов;
- `Панель` — персональная рабочая область пользователя;
- `Уведомления` — события по курсам, тестам, проверкам и апелляциям;
- `Профиль` — данные пользователя и переход к редактированию;
- `Создать курс` — быстрый переход для преподавателя;
- `Админ` — административная панель для пользователей с правами staff.

Набор пунктов зависит от того, вошел ли пользователь в систему и какая у него роль.

## Как работает сценарий студента

1. Студент входит в систему.
2. Открывает раздел `Курсы`.
3. Выбирает нужный курс и записывается на него.
4. Открывает страницу курса и смотрит доступные тесты.
5. Нажимает запуск теста.
6. Отвечает на вопросы. Во время прохождения ответы сохраняются в черновик.
7. Отправляет попытку.
8. Открывает результат: процент, набранные баллы, статус прохождения и разбор ответов.
9. При несогласии с результатом отправляет апелляцию.
10. Следит за новыми событиями в разделе `Уведомления`.

У тестов могут быть ограничения: время прохождения, количество попыток, период доступности и проходной балл. Если преподаватель выдал индивидуальные условия, студент может получить дополнительное время или дополнительные попытки.

## Как работает сценарий преподавателя

1. Преподаватель входит в систему.
2. Создает курс или открывает уже существующий.
3. Заполняет описание курса, учебный год, семестр и политику оценивания.
4. Создает тест внутри курса.
5. Указывает настройки теста: лимит времени, проходной балл, число попыток и доступность.
6. Добавляет вопросы и варианты ответов.
7. Публикует объявления для студентов курса.
8. Просматривает завершенные попытки и результаты.
9. При необходимости оставляет комментарий к попытке.
10. Открывает аналитику курса, чтобы увидеть средний результат, слабые темы и студентов, которым нужно внимание.
11. Рассматривает апелляции студентов.

Преподаватель видит только те учебные материалы, которыми он управляет. Это помогает не смешивать данные разных курсов.

## Тесты, попытки и результаты

Тест состоит из вопросов. Вопросы могут иметь один или несколько правильных вариантов ответа. У каждого вопроса есть баллы, тема и уровень сложности.

Когда студент начинает тест, создается попытка. В попытке фиксируются:

- студент;
- тест;
- время начала;
- лимит времени на момент запуска;
- выбранные ответы;
- итоговые баллы;
- процент результата;
- статус прохождения.

После отправки приложение автоматически проверяет ответы и рассчитывает итог. Если результат равен проходному баллу или выше него, попытка считается успешной.

## Уведомления и апелляции

Уведомления помогают не пропустить важные события:

- новое объявление;
- изменение по тесту;
- результат попытки;
- комментарий преподавателя;
- решение по апелляции.

Апелляция нужна, если студент считает, что результат попытки требует пересмотра. Студент отправляет сообщение по завершенной попытке, а преподаватель принимает или отклоняет апелляцию и оставляет ответ.

## Аналитика для преподавателя

Раздел аналитики помогает быстро понять состояние курса:

- какие темы вызывают сложности;
- какой средний результат у студентов;
- кто показывает слабую динамику;
- какие попытки выглядят подозрительно;
- какие тесты требуют доработки.

Также преподаватель может экспортировать результаты курса в CSV-файл для дальнейшей обработки.

## API и Swagger

Кроме web-интерфейса, у приложения есть REST API. Это полезно для демонстрации backend-части, интеграций и автоматизированных сценариев.

После запуска проекта доступны:

```text
Swagger UI: http://127.0.0.1:8000/api/docs/
OpenAPI schema: http://127.0.0.1:8000/api/schema/
```

В папке `docs` также лежат Postman-коллекция и сценарий демонстрации API.

## Частые вопросы

**Почему я не вижу кнопку создания курса?**

Создавать курсы может только пользователь с ролью преподавателя.

**Почему тест нельзя запустить?**

Возможные причины: тест не опубликован, истек период доступности, студент не записан на курс или закончились попытки.

**Где посмотреть свой результат?**

После отправки попытки приложение открывает страницу результата. Также к результатам можно вернуться через страницу курса, теста или рабочую панель.

**Что делать, если студенту нужно больше времени?**

Преподаватель может выдать индивидуальные условия по конкретному тесту: дополнительное время или дополнительные попытки.

**Где управлять всеми данными сразу?**

Для административного управления используется панель `/admin/`.

## Короткий маршрут для демонстрации

1. Запустить проект и загрузить демо-данные.
2. Войти как преподаватель `teacher_demo`.
3. Показать курс, тесты, вопросы, аналитику и попытки.
4. Выйти и войти как студент `student_demo`.
5. Открыть каталог курсов, пройти тест и посмотреть результат.
6. Подать апелляцию.
7. Вернуться под преподавателем и показать рассмотрение апелляции.
